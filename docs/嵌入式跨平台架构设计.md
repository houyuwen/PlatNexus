1. 核心架构思想
系统的核心诉求是实现 100% 的业务代码跨平台，使得核心逻辑能够在 FreeRTOS 等强实时环境中高效运行，同时能够在 Linux 环境下无缝编译用于仿真和单元测试。
实现这一目标的唯一准则是：隔离业务逻辑与底层实现。业务代码禁止直接 #include 任何具有平台属性的头文件（如 <pthread.h> 或 FreeRTOS.h）。

2. 操作系统抽象层设计
由于系统以 RTOS 为主战场，放弃使用笨重且存在语义碎片化的原生 POSIX 接口，采用自定义的极简操作系统抽象接口。

接口设计： 采用“不透明指针”（Opaque Pointers，即 void*）隐藏底层句柄（如 pthread_t 或 TaskHandle_t）。

数据类型： 强制使用 <stdint.h> 的标准定长类型，禁用系统相关的 int、long。

Linux 角色： 在 Linux 环境下，将操作系统抽象接口接口（如 plat_thread_create）底层包装为 POSIX 接口，仅作仿真运行之用。

资源映射： 操作系统抽象接口需统一调度参数（如以“字节”为单位的栈大小、统一的枚举优先级），在底层适配文件（如 plat_linux.c, plat_freertos.c）中完成平台差异的换算。

3. 硬件抽象层设计
为了彻底规避类似 POSIX ioctl 接口带来的松散类型和平台碎片化陷阱，硬件操作采用强类型的面向对象设计。

配置结构化： 为每种外设定义全平台统一的强类型配置结构体（如 plat_uart_cfg_t，包含波特率、停止位等）。

生命周期： 提供类似文件的生命周期管理，但使用自定义句柄（如 plat_uart_t），包含 open, read, write, close。

设备映射机制： 接口层的 open 函数传入设备字符串名称（如 "uart1"），在 HAL 具体实现内部使用注册表/映射表，将其转换为具体的硬件实例（如 STM32 的 USART2 或 Linux 的 /dev/ttyS1）。

4. 异步事件与中断 (ISR) 处理设计
跨平台中最难的上下文割裂（RTOS 的物理中断 vs Linux 的用户态线程）通过“硬件抽象层回调 + 操作系统抽象层延迟处理”模式解决。

回调上下文传递： 硬件抽象层定义的所有异步回调函数签名必须包含 void *used 参数，用于将业务上下文对象透传回业务层。

禁止 ISR 业务逻辑： 业务层收到的回调必须假定其运行在严格的 ISR 上下文中。回调内严禁执行业务逻辑、阻塞或延时。

跨平台唤醒机制： 操作系统抽象层提供专门用于中断的唤醒 API（如 plat_event_set_from_isr）。回调函数仅负责调用此 API 唤醒后台业务线程。

Linux 侧的模拟： 在 Linux 环境下实现硬件抽象层时，若遇到硬件中断需求（如 GPIO 跳变），硬件抽象层内部需悄悄创建一个后台 POSIX 线程，通过 epoll 等机制阻塞监听，触发后直接调用回调函数，完美欺骗业务层。
