# PlatNexus 跨平台嵌入式架构构建与组织指南

## 1. 生产级跨平台 SDK 目录结构

**PlatNexus** 的核心设计原则是：**收拢平台入口、解耦 OS 与硬件、严格隔离软件算法/硬件芯片/板级管脚与策略差异**。通过深度路径隔离，确保不同生命周期的代码彻底解耦。

## 2. 大分层路径的核心定位与解耦设计理由

### 2.1 板级支持层 (`boards/`) —— 软硬件解耦的桥梁

**核心定位：** 负责隔离特定芯片与纷繁复杂的电路板硬件变体差异。其职责可高度浓缩为 **“四定一初”**：

* **引脚定轨：** 定义片上外设（UART/I2C/SPI等）到具体电路板物理引脚的映射与电气属性。
* **时钟定频：** 配置板载外部晶振源、内部锁相环（PLL）倍频及外设总线分频系数。
* **外设定界：** 声明板载外部挂载的衍生器件（如传感器、外扩存储）的通信总线归属及具体地址。
* **内存定址：** 划分内外部 RAM/Flash 的物理地址分区（Bootloader、App、KV数据库）。
* **板级初启：** 承载该特定硬件上电时特有的早期电源管理（PMIC）配置、特定外设的硬件低电平复位脉冲时序等逻辑。

### 2.2 平台抽象层 (`platform/`) —— 抹平软硬件行为差异

**核心定位：** 仅管理特定芯片系列的通用外设控制器行为（如 STM32 或是 ESP32 的寄存器/HAL库行为）以及底层操作系统（如 FreeRTOS、POSIX）的系统调用转化。

* **hal/：** 内部代码不含有任何关于电路板引脚引脚号的硬编码。它只负责实现抽象方法，所需的物理资源全部在运行时通过设备名称或强类型描述结构体进行查找和注入。
* **osal/：** 统一提供 `plat_thread_create` 等具有全局统一表现行为的接口。
* **fs/ & net/（文件与网络系统路径归宿）：** 它们在底层深度依赖特定的操作系统或硬件协议栈。因此，它们属于**平台依赖层**的一部分，对外包装出统一的 `plat_fs_open()` 和 `plat_socket_create()`。

### 2.3 衍生器件驱动层 (`platform/drivers/`) —— 统一型号管理

**核心定位：** 专门解决如温湿度传感器等外设多型号、多驱动的共存痛点。

* 它对上层（`services`、`core_app`）提供统一的设备驱动模型 API（如 `plat_sensor_th_read`）。对下则容纳 `sht30`、`aht20` 等多型号驱动实现。
* **核心依赖准则：** 其具体的型号驱动在操作物理硬件时，**绝对禁止**越级调用芯片原生的厂商 HAL 库。必须强制调用平台层提供的 `plat_i2c_write` 等抽象接口，从而实现驱动代码本身具备 100% 芯片级无关性。

### 2.4 中间件服务层 (`services/`) —— 通用无状态软件总线

**核心定位：** 存放如控制台（Console）、日志（Log）等**平台无关、但依赖抽象接口**的通用组件。

* 它们本身是 100% 跨平台的（单片机与 Host 上完全共用一套服务逻辑代码），所以**绝对不能放进 `platform/` 内部**。
* **控制台合并（Log 与 Shell 强并合）：** 将原有的日志与调试组件统一降级归入 `services/console/` 内部。在内部由核心模块 `console_core` 接管并独占底层的某个 `plat_uart` 句柄。通过内部封装的系统级互斥锁，彻底杜绝了多线程环境下抢占物理串口导致数据碎片交织错乱的传统顽疾。

### 2.5 纯软件工具层 (`utils/`) —— 绝对纯净的算法库

**核心定位：** 它是整个系统架构依赖拓扑图的最底层（叶子节点）。

* 内部只容纳纯数学与逻辑代码（位操作、数据结构环形缓冲区、大小端互换、CRC 校验）。它**不依赖任何寄存器，也不依赖任何 OS 接口**，可在单片机和 Host 仿真环境无缝自由运行。

## 4. 无污染、强安全的板级管理实现 (`boards/`)

为了既能享受强类型枚举和面向对象描述结构体的静态安全检查优点，又能保证头文件极端隔离（不向上污染厂商 HAL 头文件），**PlatNexus** 强制将厂商寄存器级的数据结构**彻底封锁隔离在 `board_cfg.c` 内部**。对外的 `board_cfg.h` 仅暴露绝对纯净的、平台无关的抽象资源查找接口。

### 4.1 彻底纯净的板级配置中心头文件 (`boards/board_stm32_iot_v1/board_cfg.h`)

* 没有任何厂商特定宏和类型的影子，任何平台均能安全无伤地包含此文件。

```c
#ifndef BOARD_CFG_H
#define BOARD_CFG_H

#include <stdint.h>
#include <stddef.h>
#include "stm32f4xx_hal.h"

/**
 * @brief 强类型枚举：显示声明该板卡所拥有的所有硬件物理引脚轨
 * 通过这种方式，编译器在数组初始化时会自动检查重复编号，避免引脚分配冲突
 */
typedef enum {
    BOARD_PIN_CONSOLE_TX = 0,
    BOARD_PIN_CONSOLE_RX,
    BOARD_PIN_SHT30_SCL,
    BOARD_PIN_SHT30_SDA,
    BOARD_PIN_MAX /* 引脚总数边界校验 */
} board_pin_id_t;

/**
 * @brief 强类型枚举：定义该电路板上分配的所有逻辑外设句柄索引
 */
typedef enum {
    BOARD_DEV_CONSOLE_UART = 0,
    BOARD_DEV_TH_SENSOR_I2C,
    BOARD_DEV_MAX
} board_dev_id_t;

/**
 * @brief 面向对象结构体：引脚定轨电气属性描述符
 */
typedef struct {
    GPIO_TypeDef* port;       /* 物理端口（如 GPIOA） */
    uint32_t      pin;        /* 芯片物理引脚号（如 GPIO_PIN_9） */
    uint32_t      mode;       /* 复用 / 输入 / 输出 */
    uint32_t      pull;       /* 上拉 / 下拉 / 悬空 */
    uint32_t      speed;      /* 电气驱动速率 */
    uint32_t      alternate;  /* 复用功能选择（AF） */
} board_pin_desc_t;

/**
 * @brief 面向对象结构体：外设定界物理连线与属性描述符
 */
typedef struct {
    void* instance;   /* 芯片底层控制器物理实例基地址（如 USART1, I2C1） */
    uint32_t      baudrate;   /* 运行波特率 / 通信速率 */
    uint8_t       device_addr;/* 外部挂载衍生器件的物理设备地址（仅 I2C 传感器等有效） */
} board_dev_desc_t;

/* 对外统一发布硬件资源描述表导出接口 */
const board_pin_desc_t* board_cfg_get_pin(board_pin_id_t pin_id);
const board_dev_desc_t* board_cfg_get_dev(board_dev_id_t dev_id);

/* 板级早期初启接口 */
void board_early_setup(void);

#endif // BOARD_CFG_H

```

### 4.2 强类型及面向对象在 `.c` 内部的完全实现 (`boards/board_stm32_iot_v1/board_cfg.c`)

* 物理硬件长成什么样、引脚如何绑定、时钟怎么配置，作为单板的核心内聚机密，全部在这里以强类型数组收拢。

```c
#include "board_cfg.h"

/**
 * @brief 显式初始化引脚配置描述数组
 * 采用强类型数组，由于引入了枚举索引安全校验，任何引脚资源的错配均能在编译期被捕捉
 */
static const board_pin_desc_t g_board_pins[BOARD_PIN_MAX] = {
    [BOARD_PIN_CONSOLE_TX] = { GPIOA, GPIO_PIN_9,  GPIO_MODE_AF_PP, GPIO_NOPULL, GPIO_SPEED_FREQ_VERY_HIGH, GPIO_AF7_USART1 },
    [BOARD_PIN_CONSOLE_RX] = { GPIOA, GPIO_PIN_10, GPIO_MODE_AF_PP, GPIO_NOPULL, GPIO_SPEED_FREQ_VERY_HIGH, GPIO_AF7_USART1 },
    [BOARD_PIN_SHT30_SCL]  = { GPIOB, GPIO_PIN_6,  GPIO_MODE_AF_OD, GPIO_PULLUP, GPIO_SPEED_FREQ_HIGH,      GPIO_AF4_I2C1   },
    [BOARD_PIN_SHT30_SDA]  = { GPIOB, GPIO_PIN_7,  GPIO_MODE_AF_OD, GPIO_PULLUP, GPIO_SPEED_FREQ_HIGH,      GPIO_AF4_I2C1   },
};

/**
 * @brief 显式初始化外设描述数组（外设定界）
 */
static const board_dev_desc_t g_board_devs[BOARD_DEV_MAX] = {
    [BOARD_DEV_CONSOLE_UART] = { .instance = USART1, .baudrate = 115200, .device_addr = 0 },
    [BOARD_DEV_TH_SENSOR_I2C]= { .instance = I2C1,   .baudrate = 400000, .device_addr = 0x44 /* SHT30芯片物理地址 */ },
};

const board_pin_desc_t* board_cfg_get_pin(board_pin_id_t pin_id) {
    if (pin_id >= BOARD_PIN_MAX) return NULL;
    return &g_board_pins[pin_id];
}

const board_dev_desc_t* board_cfg_get_dev(board_dev_id_t dev_id) {
    if (dev_id >= BOARD_DEV_MAX) return NULL;
    return &g_board_devs[dev_id];
}

/* 时钟定频与板级初启核心动作 */
void board_early_setup(void) {
    /* 1. 配置系统时钟树（如配置主频至 168MHz） */
    SystemClock_Config_From_HSE();
    
    /* 2. 执行物理板特定的复位时序（如某些外设上电前必须对其物理引脚进行电平复位脉冲） */
    __HAL_RCC_GPIOB_CLK_ENABLE();
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_0, GPIO_PIN_RESET);
    HAL_Delay(10);
    HAL_GPIO_WritePin(GPIOB, GPIO_PIN_0, GPIO_PIN_SET);
}

## 6. 全局裁剪中心与三维 CMake 全景落地

这一部分保持原有的设计精华：通过全局策略配置 `plat_config.h` 实现软件应用控制，配合 CMake 的条件编译物理裁剪以及 `.h` 接口的桩化（Stub），实现极度闭联的安全体积缩减。

### 6.1 全局软件策略控制 (`config/plat_config.h`)

```c
#ifndef PLAT_CONFIG_H
#define PLAT_CONFIG_H

#define CONFIG_SERVICE_CONSOLE_ENABLE      1  // 是否启用控制台总线调度中心
#define CONFIG_SERVICE_LOG_ENABLE          1  // 是否开启日志打印服务
#define CONFIG_SERVICE_SHELL_ENABLE        0  // 0 代表完全裁剪并彻底移除 Shell 交互服务

#define CONFIG_CONSOLE_RX_BUF_SIZE         256 
#define CONFIG_LOG_MAX_LINE_LEN            128 

#endif // PLAT_CONFIG_H

```

### 6.2 顶层总线构建入口 (`PlatNexus/CMakeLists.txt`)

```cmake
cmake_minimum_required(VERSION 3.10)
project(PlatNexus C CXX)

set(PLAT_OS "POSIX" CACHE STRING "Target Operating System: POSIX, FREERTOS, RTTHREAD")
set(PLAT_HW "POSIX" CACHE STRING "Target MCU Silicon: POSIX, STM32, ESP32")
set(PLAT_BOARD "BOARD_POSIX_SIM" CACHE STRING "Target Board Profile: BOARD_POSIX_SIM, BOARD_STM32_IOT_V1")
set(DRV_SENSOR_TH "MOCK" CACHE STRING "Selected Sensor Model: MOCK, SHT30, AHT20")

include_directories(${CMAKE_CURRENT_SOURCE_DIR}/config)

add_subdirectory(boards)       # 优先处理单板私有数据与时钟时序
add_subdirectory(utils)        # 全平台通用零依赖数学工具
add_subdirectory(platform)     # 核心平台唯一入口大路由器
add_subdirectory(services)     # 中间件服务层
add_subdirectory(core_app)     # 100% 跨平台纯洁业务逻辑

if(PLAT_OS STREQUAL "POSIX")
    enable_testing()
    add_subdirectory(tests)    
endif()

add_subdirectory(examples)

```

### 6.3 平台核心路由器 (`PlatNexus/platform/CMakeLists.txt`)

```cmake
add_library(platform STATIC)

target_include_directories(platform PUBLIC 
    osal/include 
    hal/include
    drivers/include
    fs/include
    net/include
)

# 强行隐式链接特定电路板导出的静态私有数据实现，而不把头文件暴露给上层
target_link_libraries(platform PRIVATE board_spec)

# ==================== 维度一：操作系统 OS 路由 ====================
if(PLAT_OS STREQUAL "POSIX")
    target_sources(platform PRIVATE osal/posix/plat_thread.c osal/posix/plat_mutex.c)
    target_link_libraries(platform PRIVATE pthread)
elseif(PLAT_OS STREQUAL "FREERTOS")
    target_sources(platform PRIVATE osal/freertos/plat_thread.c osal/freertos/plat_mutex.c)
endif()

# ==================== 维度二：硬件芯片 MCU 外设路由 ====================
if(PLAT_HW STREQUAL "POSIX")
    target_sources(platform PRIVATE hal/posix/plat_uart.c hal/posix/plat_i2c.c)
elseif(PLAT_HW STREQUAL "STM32")
    target_sources(platform PRIVATE hal/stm32/plat_uart.c hal/stm32/plat_i2c.c)
    target_link_libraries(platform PRIVATE bsp_stm32)
endif()
